Next.js-Specific Checks

Verify generateMetadata() is implemented per-page, not just at layout level

Next.js App Router requires each page.tsx to export a generateMetadata() function for unique title/description per route. If only the root layout has metadata, all pages will share the same title and description.

Check each page.tsx for an exported generateMetadata function. At minimum: /, /storm-damage, /roofing, /insurance-claims, /houston, /dallas, /faq, and all /guides/* pages need unique metadata.
High

Confirm robots config in next.config.js or app/robots.ts doesn't emit noindex in production

A common Next.js deployment mistake is having NEXT_PUBLIC_NOINDEX=true or a staging robots config pushed to production. This would explain the zero indexed pages finding.

Check next.config.js → headers(), app/robots.ts, and any middleware that modifies X-Robots-Tag header. In production, the header should NOT include "noindex".

next/image is used for all images
